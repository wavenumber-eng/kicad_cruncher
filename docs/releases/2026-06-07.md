# KiCad Cruncher 2026-06-07

Release version: `2026.6.7`

This release adds a public KiCad workstation helper command for install,
process, launch, stop, and preference discovery workflows. It is intended to
support plugin setup and daemon workflows without moving that workstation
orchestration back into appz.

## Highlights

- Adds `kicad-cruncher kicad installs` to report detected KiCad install roots,
  executables, exact reported KiCad versions, and nightly/dev version folders.
- Adds `kicad-cruncher kicad running` for KiCad-family process discovery.
- Adds `kicad-cruncher kicad launch` for stable or nightly KiCad installs,
  including `--new` for starting the project manager without reloading the
  previous project.
- Adds `kicad-cruncher kicad stop` as a dry-run process report by default;
  `--all` is required before matching processes are terminated.
- Adds `kicad-cruncher kicad prefs` for config, documents, plugin directory,
  API preference, and KiCad Python interpreter diagnostics.
- Adds `kcr` as a short console alias for `kicad-cruncher`.

## Validation Notes

The launch and stop helpers were validated locally against KiCad 10.0. KiCad
source confirms that `kicad.exe` accepts project-manager file arguments for
`.kicad_pro` and legacy project files, and that `--new` prevents automatic
reload of the previous project.
