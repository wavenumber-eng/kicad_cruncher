# KiCad Cruncher 2026-06-11

Release version: `2026.6.11`

## Highlights

- Updates all generated command configs to use documented JSONC emitted from
  structured payload/comment metadata.
- Adds per-field generated comments and enum option comments for BOM/PnP/JLC,
  `pcb-svg`, `pcb clean`, and `pcb-layer-step` configs.
- Removes old config fallback behavior: BOM/PnP/JLC no longer probes
  `bom.config.json`, and `pcb-layer-step` no longer probes `pcb-layer-step.json`.
- Moves fixture STEP color/body configuration under `features.*` with
  `step_body_name`, per-feature `thickness_bias_mm`, and
  `features.component_pads.highlight_rules`.
- Adds scoped drill modes for selected component pads, other component pads,
  free pads, and vias.
- Defaults board outline/cutout STEP bodies to yellow and includes `TP*` plus
  `M*` component pads in the fixture-alignment template.
- Keeps controlled dependency pins at `kicad-monkey==2026.6.10` and
  `wn-geometer==2026.6.10`.

## Breaking Config Change

`colors`, `pad_rules`, `pad_color_rules`, and feature `body` fields are removed
from `pcb-layer-step` v2 configs. Use `features.defaults.color`, per-feature
`color`, `step_body_name`, and
`features.component_pads.highlight_rules` instead.

Public config files now must declare the current schema id. The old
`wn.kicad_cruncher.bom.config.v1` schema spelling and pcb-svg `simple` HLR mode
aliases are no longer accepted.
