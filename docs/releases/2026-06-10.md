# KiCad Cruncher 2026-06-10

Release version: `2026.6.10`

This release updates PCB SVG assembly projection to the current Geometer
projection contract and its OCCT V8-backed `2026.6.10` package.

## Highlights

- Pins `kicad-monkey==2026.6.10` and `wn-geometer==2026.6.10`.
- Uses Geometer's `outline` projection mode for assembly silhouettes.
- Defaults assembly outline projection to Geometer's `mesh-shadow` algorithm.
- Generates `ASSEMBLY_HLR_TOP_OUTLINE` and `ASSEMBLY_HLR_BOTTOM_OUTLINE`
  virtual layer tokens.
- Accepts legacy `simple` projection values and `ASSEMBLY_HLR_*_SIMPLE` tokens
  as aliases for `outline`.
- Merges pin-1 markers, board cutouts, drills, and slots into the default
  assembly top/bottom views instead of generating separate pin-1 default views.
- Reduces the generated default view list to only `assembly_top_view` and
  `assembly_bottom_view`.
- Uses Geometer `outline` mode as the default assembly HLR mode.
- Uses almost-white drill/slot overlays and bold monospace designator text in
  the default assembly views.
- Falls back to hole bounds for model-less component assembly outlines, using
  pad bounds only when a footprint has no drill or slot geometry.

## Compatibility Notes

The public PCB SVG config remains `pcb.svg.config.a0`. Existing configs that
use `simple` continue to parse, but generated configs, SVG metadata, and
manifests now use `outline`.

## Signoff

- Uses the date-versioned package contract for `2026.6.10`.
- Pins `kicad-monkey==2026.6.10` and `wn-geometer==2026.6.10`.
- CI and release workflows run Rack, build, Twine check, and installed-console
  smoke tests before publishing.
