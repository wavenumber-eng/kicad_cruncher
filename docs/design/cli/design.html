Design IndexCLI

CLI command design

design

Generates a KiCad design review bundle from KiCad project or schematic inputs.

Usage

kicad-cruncher design project.kicad_pro -o output/design
kicad-cruncher design-review project.kicad_pro
kicad-cruncher dr project.kicad_pro
kicad-cruncher design schematic.kicad_sch
kicad-cruncher design --no-indexes

Arguments

file accepts .kicad_pro or .kicad_sch. When omitted, the command attempts project auto-detection when exactly one .kicad_pro exists in the current directory. -o selects the output directory directly, and --no-indexes omits lookup indexes from the design JSON. design-review and dr are public aliases for the same command.

Output

The command writes KiCadDesign.to_json() output to <input-stem>_design.json. When -o is omitted, artifacts are written under ./output/design/. The payload is owned by kicad-monkey and includes project metadata, schematic hierarchy, sheets, components, nets, variants, optional PnP data, and optional lookup indexes.

The command also writes design_review_manifest.json, README.md, enriched black-and-white schematic SVGs under schematics/, and one PCB review SVG per copper layer under pcb/copper_layers/ when a board exists. PCB review SVGs include the selected copper layer, Edge.Cuts, and coloured drill/slot overlays. The PCB review theme is kicad_cruncher.design_review.pcb_svg.a0; draw order is tracks/arcs, polygons/zones, edge cuts, pads, then drill/slot overlays.

Tests

L3 verifies command execution against a redistributable synthetic KiCad project and copied public KiCad corpus projects. It asserts the design JSON schema, project text variables, component list, net list, index suppression behavior, alias behavior, manifest schema, README content, schematic SVG artifacts, and PCB review SVG colors, metadata, and drill/slot overlays.
