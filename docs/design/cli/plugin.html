CommandsDesign Index

Public command

plugin

Installs and inspects bundled KiCad IPC plugin packages.

Usage

kicad-cruncher plugin list-targets
kicad-cruncher plugin install --dry-run
kicad-cruncher plugin install --enable-api
kicad-cruncher plugin status
kicad-cruncher plugin uninstall --dry-run

Arguments

Subcommands accept --plugins-dir and --kicad-version to target one KiCad user plugin folder. install also accepts --dry-run, --create-default, --best-effort, --enable-api, --skip-api-setup, and --python-interpreter. The default bundled plugin package is kicad-cruncher-tools.

Output

The command prints discovered target directories, installed/uninstalled plugin paths, and KiCad IPC API preference diagnostics. Installed plugin files are copied from the public package into the user KiCad plugin directory. The bundled kicad-cruncher-tools action health-checks the local daemon, posts a PCB clean planning request, then opens the daemon tool center.

Tests

L0 verifies command registration and help. L3 verifies the daemon command inventory and plugin-mode PCB clean routing contract. Future installer tests will use temporary KiCad document and config roots to validate dry-run, copy filters, and API preference updates without touching a real user profile.
